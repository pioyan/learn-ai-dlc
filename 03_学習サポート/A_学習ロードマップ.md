# A: AI-DLC 学習ロードマップ

AI-DLC を「知っている」から「自分で動かせる」まで到達するための段階的な学習順序。

---

## 全体像：3つのフェーズ

```
フェーズ1: 理解          フェーズ2: 体験          フェーズ3: 実践
(1〜2週間)               (2〜4週間)               (継続)
─────────────────────────────────────────────────────────────
概念を掴む               手を動かす               チームで使う
    │                        │                        │
    ↓                        ↓                        ↓
「何をするのか             「どうやるのか            「現場で
  分かる」                  分かる」                 動かせる」
```

---

## フェーズ1: 理解（Week 1〜2）

### 目標
AI-DLC の「何が・なぜ・どのように」を説明できる状態になる

### ステップと順序

```
Step 1 (30分)
  └─ [A_学習ロードマップ.md] ← 今ここ
       このファイルで全体像を把握

Step 2 (30分)
  └─ [H_概念図ダイアグラム.md]
       図で構造を視覚的に掴む

Step 3 (2時間)
  └─ [01_基礎/AI-DLC_理解と実践ガイド.md]
       3分サマリー → 全体像 → 90日ロードマップを通読

Step 4 (必要に応じて)
  └─ [B_用語集.md]
       読んでいて意味が分からない言葉を確認

Step 5 (1〜2時間)
  └─ [01_基礎/AI-DLC_Method_Definition_ja.md] の I〜III 章
       原文翻訳で原理を理解

フェーズ1 完了チェック（[G_自己確認クイズ.md] の「フェーズ1クイズ」で確認）
```

### フェーズ1 で答えられるようになること
- [ ] AI-DLC の 3 つのフェーズ（Inception / Construction / Operations）を説明できる
- [ ] Intent / Unit / Bolt の違いを自分の言葉で言える
- [ ] Mob Elaboration と Mob Construction の役割分担が説明できる
- [ ] 従来の Scrum と何が違うか説明できる

---

## フェーズ2: 体験（Week 3〜6）

### 目標
実践例を読みながら、自分の手で小さな実験をする

### ステップと順序

```
Step 6 (3時間)
  └─ [02_モブワーク実践事例/実践例_01_Mob_Elaboration.md]
       一人で通読 + 自分の業務に当てはめて考える
       ※ ここでAIツールを使って、架空のシナリオで Intent を書いてみる

Step 7 (4時間)
  └─ [02_モブワーク実践事例/実践例_02_Mob_Construction.md]
       一人で通読 + コード例を実際に動かしてみる
       ※ Python環境があれば、Domain Model のコードをコピーして実行

Step 8 (1時間)
  └─ [E_プロンプト設計基礎.md]
       AIへの質問の設計原則を学ぶ
       ※ 自分でプロンプトを作って試してみる

Step 9 (状況に合わせて選択)
  └─ 実践例_03〜05 のうち、自分の業務に近いものを 1 つ読む
       03: 既存システムがある → Brownfield
       04: チームが複数ある → Microservice
       05: 金融・規制業界 → Security

フェーズ2 完了チェック（[G_自己確認クイズ.md] の「フェーズ2クイズ」で確認）
```

### フェーズ2 で答えられるようになること
- [ ] Mob Elaboration で AI へどう質問するか、具体例を挙げられる
- [ ] Intent ドキュメントを 1 件、自分のシナリオで書ける
- [ ] Unit 分解の判断基準（いつ分けて、いつ一まとめにするか）が言える
- [ ] プロンプトの「良い例・悪い例」を識別できる

---

## フェーズ3: 実践（Week 7 以降）

### 目標
小さなチームで実際に AI-DLC を試す

### ステップと順序

```
Step 10 (チームと確認)
  └─ [C_前提知識チェックリスト.md]
       チームメンバーに必要な前提知識を確認
       不足があれば補足資料を提供

Step 11 (1〜2時間のキックオフ)
  └─ [D_従来手法との比較.md]
       チームに「今までの開発と何が違うか」を説明する

Step 12 (実施)
  └─ Mob Elaboration を 1 回試す（実践例_01 を参照しながら）
       ポイント: 小さな範囲（1機能・1エピック）から始める

Step 13 (振り返り)
  └─ [F_アンチパターン集.md]
       やってみて陥った罠を照合する

継続...
  └─ DORA メトリクスで改善を数値化
     （01_基礎/AI-DLC_理解と実践ガイド.md の「5. 外部知見」参照）
```

## 学習スタイル別 推奨順序

### 読書派（ドキュメントを順に読みたい）
```
H → A → B → C → D → ガイド → 翻訳I-III → E → 実践例01 → 02 → F → G
```

### 実践派（手を動かしながら学びたい）
```
A → ガイド（3分サマリーだけ） → 実践例01を試す → 詰まったらB,E → G で確認
```

### 既存Agile経験者
```
D（違いを確認） → H（図で掴む） → 実践例01をすぐ試す → 詰まったらB
```

### マネージャー・PO（チームに導入したい）
```
A → ガイド全体 → D → C（チームの前提知識確認） → 実践例01のみ通読
```
