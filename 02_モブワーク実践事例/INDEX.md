# モブワーク実践事例集 - 完全ガイド

このディレクトリには、AI-DLCで中心的な役割を果たす「モブワーク」の5つの実践シナリオが収録されています。  
各ドキュメントは、ビジネスシナリオごとに段階的なプロセスと具体的なコード例を提供します。

---

## 実践ドキュメント一覧

### 1 [Mob Elaboration 実践例](実践例_01_Mob_Elaboration.md)

**シナリオ**: クロスセル向け推薦エンジンの要件を Intent → Unit → Bolt へ分解

**所要時間**: 4～6 時間（1 日～複数日セッション）

**参加者**: PO、テックリード、開発者、QA、ドメイン専門家

**ステップ**:
- Intent 明確化（ビジネス目標の定義）
- ユーザーストーリー生成（AI 支援）
- Unit 分解（独立構築可能な単位）
- Bolt 計画（時間～日単位の反復）

**学べること**:
- 曖昧なビジネス要件を明確に翻訳する流れ
- AI が生成した案を現実的に調整するプロセス
- チーム全員の理解を揃える技法

**対象読者**: PO、プロダクトマネージャー、テックリード

---

### 2 [Mob Construction 実践例](実践例_02_Mob_Construction.md)

**シナリオ**: 推薦エンジンコアを、Domain Design → Logical Design → Code & Test で実装

**所要時間**: 2 週間（毎日 6～8 時間のMob）

**参加者**: テックリード、シニア開発者、ジュニア開発者、QA

**フェーズ**:

**Week 1: 設計**
- Domain Model（DDD に基づく）定義
- AWS サービス選定（DynamoDB, Lambda, ElastiCache）
- Architecture Decision Records (ADR) 作成

**Week 2: 実装・テスト**
- Mob Programming でコード生成
- ユニットテスト & Integration テスト
- 負荷テスト & パフォーマンス検証

**学べること**:
- Domain Driven Design の実装方法
- AI コード生成と人間のレビューの融合
- テスト駆動開発（TDD）の実践
- AWS ベストプラクティス

**対象読者**: 開発者、アーキテクト、テックリード

### 3 [ブラウンフィールド Mob Construction](実践例_03_Brownfield_Mob.md)

**シナリオ**: 既存 e-Commerce システムに「顧客360度ビュー」機能を追加しながらモダナイゼーション

**所要時間**: 2 週間

**特徴**:
- 既存レガシーコードの逆解析
- Static Model & Dynamic Model による理解
- 段階的な AWS 移行計画（Big Bang vs 段階的）
- 後方互換性（Backward Compatibility）の保証

**学べること**:
- レガシーコード理解に AI を活用する方法
- 既存システムと新システムの共存戦略
- Canary Deploy で低リスク移行
- Oracle → AWS RDS への段階移行手法

**対象読者**: 既存システムの保守者、アーキテクト

---

### 4 [マイクロサービス Mob Construction](実践例_04_Microservice_Mob.md)

**シナリオ**: 支払い機能を Order Service から独立させ、イベント駆動で連携

**所要時間**: 2～3 週間

**特徴**:
- Domain Event Storming（Event 中心設計）
- Service Boundary の引き方
- 複数チームの並列開発調整
- Distributed Tracing & Resilience Pattern

**学べること**:
- マイクロサービスアーキテクチャ設計
- 非同期イベント駆動設計
- EventBridge / Kafka の使い分け
- サービス間の統合テスト方法

**対象読者**: アーキテクト、システムデザイナー、バックエンド開発者

### 5 [セキュリティ重視 Mob Construction](実践例_05_Security_Mob.md)

**シナリオ**: 金融機関向け「顧客資産管理 API」を ISO 27001 + PCI DSS 要件で実装

**所要時間**: 2 週間

**特徴**:
- STRIDE Threat Model を最初に作成
- Zero Trust Architecture 実装
- 暗号化・鍵管理の自動化（AWS KMS）
- 改ざん防止の監査ログ（ハッシュチェーン）
- ペネトレーションテスト & Compliance チェック

**学べること**:
- Threat Model による包括的な脅威分析
- 金融・規制業界のセキュリティ要件
- PCI DSS / ISO 27001 への対応方法
- セキュリティテストの自動化

**対象読者**: セキュリティアーキテクト、コンプライアンス担当者、開発リーダー

## シナリオ別 選択ガイド

| 状況 | 最適なドキュメント | 理由 |
|------|-----------------|------|
| 新規プロジェクト立ち上げ | 1 → 2 | 要件明確化 → 実装流れを学ぶ |
| 既存システム機能追加 | 3 | レガシー理解と並列開発の実績あり |
| マイクロサービス化 | 4 | Event 駆動設計とサービス分割を学ぶ |
| 金融・規制業界進出 | 5 | セキュリティ要件と検証方法を習得 |
| 全体を理解したい | 1 → 2 → 3 → 4 → 5 | 段階的に技術深度を上げられる |

## 各ドキュメントの構成

すべてのドキュメントは以下の構造を共通して含みます：

### 最初のセクション
- **シナリオ**: 何を作るのか（ビジネス背景）
- **所要時間**: 実施に必要な期間
- **参加者**: 誰が関わるのか
- **チャレンジ**: このシナリオ固有の難しさ

### 実行フェーズ
各フェーズは以下を含む：
- **目的**: このフェーズで何を達成するのか
- **AI プロンプト例**: 実際に使える質問文
- **AI 出力例**: 期待される結果
- **ファシリテータのモデレーション**: チーム内での調整方法
- **コード例**: 実装の具体例

### 検証・チェックリスト
- **完了条件**: このフェーズが終わったという証拠
- **よくある失敗パターン**: 陥りやすい罠
- **次のステップ**: 次は何をするのか

## ドキュメント活用方法

### 個人での学習
1. 自分の業界・状況に合ったドキュメントを選ぶ
2. "AI プロンプト例" の章をコピー
3. 実際のプロジェクトで試してみる
4. 結果を記録して改善

### チーム導入
1. **Week 0**: 該当ドキュメント全体を読み合わせ（1 時間）
2. **Week 1-2**: "実行フェーズ" に従い、実際にMobを実施
3. **Week 3**: チーム内で「うまくいった工夫」「つまずいた点」を共有
4. **Week 4+**: 次のプロジェクトで改善版を実施

### コーチ・ファシリテータ向け
1. 進行前に担当ドキュメント全体を把握
2. "ファシリテータのモデレーション" の例を参考に、チーム固有のやり方を工夫
3. チームの質問に対し、該当セクションを参照しながら答える

## AI ツール使い分けガイド

### Claude (Anthropic)
- **得意**: 複雑な設計思考、Domain Model 定義
- **用途**: 1 番ドキュメント（Elaboration）で活躍

### GitHub Copilot / Cursor
- **得意**: コード補完、実装加速
- **用途**: 2 番ドキュメント（Construction）のコード生成

### ChatGPT
- **得意**: 汎用的な知識提供、文章生成
- **用途**: セキュリティ脅威分析（5 番）の補助

**推奨**: 複数ツール組み合わせで、各フェーズで最適なツール選択

## 導入時の段階的アプローチ

### Phase 1: パイロット（1 ヶ月）
- 1 つのドキュメント（最も関連度高い）を選択
- 1 チームで実施
- 学習・改善に注力

### Phase 2: 検証（1 ヶ月）
- パイロット結果を共有
- 別のチームで 2 番目のドキュメントを試す
- 共通パターンを抽出

### Phase 3: スケール（継続）
- 組織標準として導入
- 全ドキュメント活用
- 社内ナレッジ化

## よくある質問

**Q1: Mob Elaboration だけでなく、Mob Construction も必須か？**  
A: 新規開発ならどちらも推奨。既存システム改善なら 3 番を優先。

**Q2: すべてのドキュメントを順番に実施する必要がある？**  
A: いいえ。現在の状況に最も近いドキュメントから始めることをお勧めします。

**Q3: チームメンバーがAIツール初心者の場合は？**  
A: 1 週間前に AI ツール基礎研修を実施してから、各ドキュメントを開始してください。

**Q4: Mob セッション中にうまくいかなかったら？**  
A: 各ドキュメントの「よくある失敗パターン」セクションを参照。多くは既知の問題です。
